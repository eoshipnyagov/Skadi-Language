## Skadi Language Specification v1.1

### 1. Философия и основные принципы
- **Человекочитаемость:** код должен читаться как обычный текст, минимум символов-«каракулей».
- **Программист не дурак:** язык не запрещает, а предупреждает. Компилятор — помощник, а не надзиратель.
- **Всё под рукой:** базовые вещи доступны без импорта, неиспользуемое не попадает в бинарник.
- **Высокая производительность:** предсказуемое управление памятью через чанки (регионы), отсутствие скрытых аллокаций и сборщика мусора.
- **Контроль над памятью без боли:** бюджет чанка, статический анализ, `allow drop` для некритичных данных.

### 2. Модель памяти (чанки)
- Память управляется **чанками (аренами)**. Каждая задача (функция, запущенная через `run`) получает отдельный чанк.
- Чанк живёт, пока выполняется задача. При завершении задачи память освобождается целиком (O(1)).
- **Бюджет чанка:** определяется статическим анализом + динамический рост + опциональный ручной лимит (`budget` в конфиге задачи).
- **Вытеснение:** переменные, помеченные `allow drop`, могут быть автоматически удалены при нехватке памяти в чанке (начиная с хвоста).
- Внутри чанка статические переменные размещаются в структуре, динамические (`List`, `String` переменной длины) растут из хвоста чанка.

### 3. Базовый синтаксис
- **Разделители:** точки с запятой отсутствуют. Одна операция — одна строка.
- **Блоки:** фигурные скобки `{ }`.
- **Условия и циклы:** без круглых скобок вокруг условий: `if x > 0 { }`.
- **Вызов функций:** со скобками: `output("Hi")`.
- **Комментарии:** `//` строчный, `/* */` блочный.
- **Строковая интерполяция:** `"Привет, __name__!"` — подстановка значения переменной `name`.

### 4. Типы данных (ядро)

#### Примитивные типы
| Тип | Описание |
|-----|----------|
| `i8`, `i16`, `i32`, `i64` | Целые со знаком |
| `u8`, `u16`, `u32`, `u64` | Целые без знака |
| `f32`, `f64` | Числа с плавающей точкой |
| `bool` | `true` / `false` |
| `char` | Unicode-символ |
| `String` | Экономная UTF-8 строка |
| `Text` | Индексируемая строка (для UI) |
| `Path` | Путь к файлу/директории |

**Алиасы:** `Int` (→ `i64`), `Float` (→ `f64`), `Byte` (→ `u8`) и др.

#### Составные типы
- `[T; N]` — статический массив фиксированной длины.
- `List(T)` — динамический список (гомогенный).
- `Vec2`, `Vec3`, `Vec4` — векторы с поэлементной арифметикой.
- `Link(T)` — типизированный канал для межзадачной связи.

**Приведение типов:**
- Неявное: безопасные расширения (Int → Float, u8 → u16).
- Явное: `Тип(выражение)` — конструктор типа, например `Int(x)`, `Float(42)`. Для String → Int требует обработки ошибок (`on error`), т.к. может упасть.

### 5. Операторы (в порядке убывания приоритета)

| Приоритет | Операторы | Описание |
|-----------|-----------|----------|
| 1 | `()` `[]` `.` `my` | Группировка, индексация, доступ к полю, доступ к полю структуры |
| 2 | `not` `-` (унарный) | Логическое отрицание, унарный минус |
| 3 | `^` | Возведение в степень |
| 4 | `*` `/` `div` `mod` | Умножение, деление, целочисленное деление, остаток |
| 5 | `+` `-` | Сложение, вычитание |
| 6 | `==` `!=` `>` `<` `>=` `<=` | Сравнения |
| 7 | `and` | Логическое И |
| 8 | `or` `xor` | Логическое ИЛИ, исключающее ИЛИ |
| 9 | `=` `+=` `-=` `*=` `/=` | Присваивание и сокращённые присваивания |

- Инкремент `++`, декремент `--` допустимы только как отдельная инструкция (не в выражениях).

### 6. Переменные и область видимости

**Объявление:**
```
x = 5               // вывод типа (Int)
Int y = 10          // явный тип
```

**Присваивание:**
```
x = 20
x += 5
```

- Область видимости блочная (`{ }`).
- Затенение переменных разрешено. Для доступа к полям текущей структуры используется префикс `my` (см. раздел Структуры).
- Константы: `fixed MAX = 100` (альтернативное ключевое слово `const`).

### 7. Функции

**Объявление:**
```
// Одно возвращаемое значение
fn add(Int a, Int b) Int {
    return a + b
}

// Множественный возврат через структуру
fn divide(Int a, Int b) returns struct {Int q, Int r} {
    return {q = a div b, r = a mod b}
}

// Опасная функция (может завершиться ошибкой)
danger fn parse(String s) Int { ... }
```

- Параметры: C-style — тип перед именем (`Int a, Float b`).
- Возвращаемый тип: если одно значение — просто тип после скобок; если несколько — `returns struct { ... }`.
- `direct` параметр: передача по ссылке (изменения видны снаружи) — `fn modify(direct Player p)`.
- Функции могут вызываться до объявления (порядок не важен).
- Локальные функции (не видны за пределами модуля): `local fn helper() { ... }`.

### 8. Структуры

```
struct Player {
    String name        // поле открыто (по умолчанию)
    hide Int hp        // поле скрыто
    hide Int x, y      // скрытые поля

    fn attack(Monster target) {
        my.hp = my.hp - target.damage   // доступ к своему полю через my
    }
}
```

- Доступ к полям внутри методов: `my.field` (аналог `self`/`this`).
- Инкапсуляция: по умолчанию всё открыто (`pub`), для сокрытия используется `hide`. Для модулей аналогично: `local fn` скрывает функцию.
- Создание экземпляра: `Player{name = "Hero", hp = 100, x = 0, y = 0}`.
- Нет наследования, композиция через встраивание.

### 9. Управляющие конструкции

#### Ветвление
```
if условие {
    // then
} else if другое_условие {
    // else if
} else {
    // else
}
```

#### Выбор (сопоставление)
```
when выражение {
    is значение1 {
        ...
    }
    is значение2, значение3 {
        ...
    }
    else {
        ...
    }
}
```

#### Циклы
```
while условие { }
for элемент in коллекция { }
loop { }  // бесконечный цикл
```

#### Обработка ошибок
Вызов `danger fn` должен сопровождаться блоком `on error`:
```
danger_fn() on error {
    // действия при ошибке
}

x = parse(str) on error {
    x = 0
}
```
- Механика: `danger fn` возвращает `Result(T, error)`. `on error` — синтаксический сахар для проверки флага ошибки. При ошибке выполняется блок, иначе результат присваивается.

#### Прерывания (embedded)
```
on interrupt timer1 {
    gpio.toggle(led)
}
```

#### События (GUI/игры)
```
on event {
    is gfx.KeyPress(Key_Escape) { gfx.close() }
    is gfx.MouseClick(x, y) { handle_click(x, y) }
}
```

### 10. Конкурентность и межзадачная связь

- Запуск задачи: `run task()`, `task_id = run task()`.
- Ожидание завершения: `result = wait task_id`.
- Задержка: `delay(500ms)`, `delay(2s)`, `delay(10us)` — задача приостанавливается, другие работают.
- Литералы времени: `500ms`, `2s`, `10min` и т.д.
- Каналы (`Link`):
  ```
  pipe = Link(Int)
  pipe.send(42)
  x = pipe.receive()
  pipe.signal()
  on pipe.signal { ... }
  ```

### 11. Ввод-вывод и стандартная библиотека

**Ядро (доступно без импорта):**
- `output(x)` (алиас `print`), `input(prompt)`.
- `write(stream, data)`, `read(stream)`.
- Файловая система: `fs.read(path)`, `fs.write(path, data)`, `fs.list(dir)`, `fs.is_dir(path)`.
- Математика: `sin`, `cos`, `sqrt`, `abs`, `rand`, `PI`, `E` и др.
- Работа с железом: `gpio`, `timer`, `uart`, `i2c`, `spi`, `adc` (при целевом embedded).
- `delay`, `run`, `wait`, `sleep(время)`, `Link`.
- `gfx`, `ui`, `term` — базовая графика и UI (Immediate Mode GUI).

**Стандартная библиотека (подключается через `import`):**
- `net` — HTTP, TCP, UDP.
- `json` — парсинг и сериализация JSON.
- `time` — форматирование дат.

**Импорт:**
```
import "path/to/file"
import module_name
import long_name as alias
```

### 12. Перечисления
```
label Status {
    Ok
    Warning
    Error
}
```

### 13. Тестирование
- Встроенная команда `compiler test модуль` генерирует файл `_test.lang` с заготовками тестов.
- Ключевое слово `test`, проверка через `check` (альтернативы `expect`, `verify`).
```
test "сложение работает" {
    check(add(2, 3) == 5)
}
```

### 14. Пример полного кода (десктопная утилита tree)
```
danger fn main() {
    root = Path(".")
    output(root)
    print_tree(root, "")
}

fn print_tree(Path dir, String prefix) {
    entries = fs.list(dir) on error {
        output("Ошибка чтения: __dir__")
        return
    }
    
    count = len(entries)
    i = 0
    
    for entry in entries {
        is_last = (i == count - 1)
        full_path = Path("__dir__/__entry__")
        
        if is_last {
            output("__prefix__└── __entry__")
        } else {
            output("__prefix__├── __entry__")
        }
        
        if fs.is_dir(full_path) {
            if is_last {
                print_tree(full_path, "__prefix__    ")
            } else {
                print_tree(full_path, "__prefix__│   ")
            }
        }
        
        i = i + 1
    }
}
```

### 15. Грамматика (выдержка для парсера)

Ниже дана упрощённая грамматика в нотации EBNF для ключевых конструкций:

```
program        = { top_level_decl } ;
top_level_decl = fn_decl | struct_decl | label_decl | import_decl | var_decl | test_block ;

import_decl    = "import" ( string_literal | ident [ "as" ident ] ) ;
fn_decl        = ["danger"] "fn" ident "(" [ param_list ] ")" return_type block ;
param_list     = param { "," param } ;
param          = ["direct"] type ident ;
return_type    = "returns" struct_literal | type ;   // упрощённо
struct_decl    = "struct" ident "{" { struct_member } "}" ;
struct_member  = ["hide"] ( var_decl | fn_decl ) ;
label_decl     = "label" ident "{" ident { "," ident } "}" ;
var_decl       = ["allow drop"] ["fixed"] type ident "=" expr | ident "=" expr ; // с выводом типа
block          = "{" { stmt } "}" ;
stmt           = var_decl | assignment | if_stmt | when_stmt | while_loop | for_loop | loop_stmt
               | return_stmt | expr_stmt | on_error_block | on_interrupt | on_event ;
assignment     = expr ( "=" | "+=" | "-=" | "*=" | "/=" ) expr ;
if_stmt        = "if" expr block [ "else" "if" expr block ] [ "else" block ] ;
when_stmt      = "when" expr "{" { when_case } [ "else" block ] "}" ;
when_case      = "is" expr_list block ;
while_loop     = "while" expr block ;
for_loop       = "for" ident "in" expr block ;
loop_stmt      = "loop" block ;
return_stmt    = "return" [ expr ] ;
expr_stmt      = expr ;
on_error_block = "on error" block ;
on_interrupt   = "on interrupt" ident block ;
on_event       = "on event" "{" { event_case } "}" ;
event_case     = "is" event_type [ "(" params ")" ] block ;

expr           = assignment_expr ;
// выражения с приоритетом стандартные, операторы см. таблицу
literal        = int_literal | float_literal | string_literal | bool_literal | struct_literal ;
struct_literal = "{" [ field_init { "," field_init } ] "}" ;
field_init     = ident "=" expr ;
```

Эта грамматика покрывает базовые конструкции, парсер можно строить методом рекурсивного спуска, для выражений — Pratt-парсер.
