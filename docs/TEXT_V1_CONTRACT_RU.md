# Skadi v1: Контракт `Text`

Дата: 2026-05-25  
Статус: зафиксированный контракт для `v1`.

## 1. Модель данных

- `Text` в `v1` хранится как UTF-8 строка.
- Все базовые операции `Text` в runtime трактуют строку как **последовательность байтов**, не графем.

## 2. Операции и семантика

1. `len(t)`
- Возвращает длину в байтах (`strlen` в C lowering), не количество Unicode-символов.

2. `t[i]`
- Индексация по байтовому смещению.
- Тип результата: `char`.
- Out-of-range: `'\0'` (fail-soft контракт `v1`).

3. `slice(t, start, end)`
- `start/end` интерпретируются как байтовые индексы.
- Границы нормализуются:

  - `start < 0 -> 0`
  - `end < start -> start`
  - `start/end > len -> len`
- Возвращает новый `Text`.

4. `find(t, needle)` / `contains(t, needle)`
- Поиск по байтовым подпоследовательностям.
- `find` возвращает байтовую позицию или `-1`.

## 3. Пример с UTF-8

Для `t = "Пр"`:

- `len(t) == 4` (два кириллических символа по 2 байта каждый в UTF-8),
- `t[0]` и `t[1]` — байты первого символа, а не отдельные Unicode-кодпоинты.

## 4. Границы v1

- `v1` не гарантирует grapheme-aware операции.
- Для UI/локализации и «символьной» семантики нужен отдельный слой (`v2+`).
