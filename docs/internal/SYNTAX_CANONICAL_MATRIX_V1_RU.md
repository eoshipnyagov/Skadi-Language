# Skadi v1: Каноника синтаксиса

Дата: 2026-05-26
Статус: заморожено для `v1`.

## Формат статусов

- `Canonical` — рекомендуемая форма в документации и showcase.
- `Allowed` — поддерживается для совместимости, но не приоритет по стилю.
- `Deferred` — отложено за рамки v1.

## Базовые правила

- Объявление переменных: только через `new` (`Canonical`).
- Цикл коллекций:
  - `iterate <collection> as <item>` (`Canonical`)
  - `for <item> in <collection>` (`Allowed`).
- Инкремент/декремент:
  - `i++`, `i--` только как отдельные инструкции (`Canonical`).
- Поток управления:
  - `break`, `continue`, `pass` поддержаны.
- Импорт модулей v1:
  - только `import "./relative_path.skd"` (`Canonical`).

## Типы: соглашение по именованию

- Низкоуровневые фиксированные: `i8/i16/i32/i64`, `u8/u16/u32/u64`, `f32/f64`.
- Высокоуровневые: `Int`, `Float`, `Bool`, `Char`, `Text`, `Path`, `List`.

## Что отложено

- Alias/import by name.
- Расширенные visibility-правила модулей.
- Полная chunk-memory модель.
