# Skadi v1: Контракт памяти C-runtime

Дата: 2026-05-26
Статус: зафиксировано для `v1` (`Skadi -> C`).

## Цель

Фиксирует правила памяти для runtime helper-функций, которые генерируются в C.

## Базовые правила v1

- Выделение памяти выполняется через `malloc/realloc/free` в helper-слое C.
- Для `List`:
  - динамический буфер растет по мере `push`,
  - `pop` уменьшает логический размер, но не обязан сжимать capacity,
  - освобождение происходит в явно сгенерированных cleanup-путях.
- Для `Text`:
  - строковые helper-операции выделяют/копируют буферы по необходимости,
  - срезы и конкатенации используют owned C-строки.

## Ошибки памяти

- При `malloc/realloc == NULL` runtime helper возвращает fail-сигнал по текущему контракту операции.
- Для операций, обернутых в `on error`, отказ переводится в ожидаемую ветку обработки.

## Что не входит в v1

- Полная chunk/arena-модель из долгосрочного дизайна языка.
- Глобальный allocator policy и тонкие оптимизации размещения.

## Инварианты для тестов

- Нет use-after-free в генерируемых позитивных сценариях.
- Нет double-free в cleanup путях.
- `List`/`Text` e2e-сценарии стабильно компилируются и исполняются на host toolchains.
