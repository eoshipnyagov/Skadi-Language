# Skadi v1: Матрица `on error`

Дата: 2026-05-26
Статус: зафиксированный контракт для `v1`.

## Разрешено в v1

1. Вызов `danger fn`:
```skadi
new Int x = parse_num(s) on error {
    x = 0
}
```

2. `List.pop()`:
```skadi
new Int v = items.pop() on error {
    v = -1
}
```

## Запрещено в v1

- `on error` для произвольных выражений, не имеющих error-контракта.
- `on error` после обычного `fn`, который не объявлен как `danger`.

## Диагностика

Ошибка должна явно отвечать на 3 вопроса:
- что именно не разрешено,
- какой контракт требуется,
- как исправить (например, `danger fn` или удаление `on error`).

## Планы 1.x

- Рассмотреть расширение `on error` на дополнительные builtin-операции,
- только после формальной матрицы: операция -> код ошибки -> lowering в C.
