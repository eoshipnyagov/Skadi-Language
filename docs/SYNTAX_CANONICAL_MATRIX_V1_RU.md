# Skadi v1: Каноника синтаксиса

Дата: 2026-05-25  
Статус: заморожено для `v1`.

## 1. Формат статусов

- `Canonical`: рекомендуемая форма в документации/витринных примерах.
- `Alias`: допустимо в `v1`, но не является каноникой.
- `Forbidden`: не поддерживается как валидная форма `v1`.

## 2. Таблица каноники

| Область | Canonical | Alias | Forbidden (v1) |
|---|---|---|---|
| Объявление переменной | `new x = expr`, `new Type x = expr` | - | формы без `new` для объявления |
| Цикл по коллекции | `iterate collection as item` | `for item in collection` | C-style `for(init;cond;step)` |
| Типы общего уровня | `Int`, `Float`, `Bool`, `Char`, `Text`, `Path` | `bool`, `char` | произвольные искажения (`INT`, `boolean`, etc.) |
| Fixed-width типы | `i8/i16/i32/i64`, `u8/u16/u32/u64`, `f32/f64` | - | варианты с иным регистром (`I32`, `U64`, `F32`) |
| Возврат ошибки | `return error Code` (только в `danger fn`) | - | `return error` вне `danger fn` |
| Обработка ошибок | `... on error { ... }` для `danger fn` и `List.pop()` | - | `on error` для builtins/индексации/произвольных выражений |
| Индексация | `xs[i]`, `t[i]` | - | `on error`-обертка над индексацией в `v1` |
| Доступ к полям структуры | `my.field` в методе структуры | - | `my` вне метода структуры |
| Выбор | `when/is/else` | - | альтернативные непарсируемые формы |

## 3. Связь с style-check

`semantic_style_warnings` в текущем `v1`:
- предупреждает, что в showcase-стиле предпочтительно `iterate ... as ...` вместо `for ... in ...`;
- предупреждает, что предпочтительны `Bool/Char` вместо `bool/char`;
- оставляет алиасы (`for`, `bool`, `char`) валидными, но неканоничными.

## 4. Правило изменения после v1

Любое изменение каноники после `v1` должно:
1. обновить эту таблицу;
2. обновить связанный статус в `SKADI_SYNTAX_STATUS.md`;
3. иметь тесты на parser/semantic/style behavior.
